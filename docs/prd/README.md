## How to use this folder

- **Before any new build in this project domain:** read the matching PRD section, review the matching concept screen, and check the build state table above.
- **When the user provides a new artifact:** save it here using the naming convention below, then update this README.
- **When something ships:** update the Status column in the build state table.
- **When the PRD or concepts change:** replace the file in place — do not add versioned copies.
- **Concept screens are not pixel specs.** They are scope signals. Real designs should come from a Figma file or approved UX before locking screen behavior.

## Naming convention

| Artifact type | Save as |
|---|---|
| PRD or requirements doc (markdown) | `<descriptive-name>.md` |
| PRD or requirements doc (PDF) | `<descriptive-name>.pdf` |
| Screenshot or UX reference | `<NN>-<descriptive-name>.png` (numeric prefix for flow order) |
| Fit-gap or org analysis | `fit-gap-<domain>.md` |
| Concept mocks | `concepts/<descriptive-name>.png` |
